# Marketplace UI/UX — the decisions

Every axis from `design.loop`, decided before code. `site.loop` implements this file.

## layout
**Decision:** responsive card grid — `repeat(auto-fill, minmax(360px, 1fr))`; 3 columns on desktop, 2 on tablet, 1 under 720px. Cards equal-height per row.
**Rationale:** loops are scannable one-liners + a code payload; a grid maximizes concepts-per-viewport, and 360px fits the longest `done when` line in Ubuntu Mono 13px without wrapping twice.

## card
**Decision:** anatomy top-to-bottom — category badge + shape glyph (↻ loop / ▶ pipeline / → flow) on one row · title (18px, 700) · goal line (muted, 2-line clamp) · `done when` first predicate in mono teal (1-line, ellipsis) · footer: `concept via <author>` (small, muted) + [Copy] [View] buttons. **No votes in v1** — no backend; leave no placeholder UI (dead chrome reads as broken).
**Rationale:** the `done when` line IS the differentiator vs Forward Future — it earns card real estate. Attribution visible on every card honors the source library.

## discovery
**Decision:** client-side search over title + goal + author + keywords (from catalog.json); category chips All/Engineering/Evaluation/Design/Operations/Content with counts; sort fixed: category → title A-Z. A live "N loops" count under the search box.
**Rationale:** 29 items — client-side is instant; fixed sort keeps browsing stable until votes exist to rank by.

## detail view
**Decision:** inline expand — [View] toggles the card open to the full `.loop` source in a `<pre>` (loopflow syntax colors), plus the use-when note and a link to the source concept. No per-loop pages in v1.
**Rationale:** one page = one deploy artifact; per-loop pages are an SEO project for phase 2 (FF has 30 crawl pages — that's their moat to copy later, not launch-blocking).

## copy action
**Decision:** [Copy] puts the raw `.loop` file text (attribution header included) on the clipboard; button text flips to "✓ copied" for 1.2s. Keyboard: the button is a real `<button>`, Enter/Space native.
**Rationale:** header travels with the copy → attribution survives paste; the whole point of the market is paste-and-adapt.

## submit
**Decision:** "＋ Submit your loop" — a header-right button and a bottom CTA card, both linking to GitHub's new-file screen prefilled to `marketplace/loops/` on a fork, plus a "read the bar" link to `marketplace/CONTRIBUTING.md`. The page states the bar in one line: *"parses + a machine-checkable done-when + attribution — CI enforces it."*
**Rationale:** uploads are PRs (the storage decision); the button makes the PR path feel like a form, and stating the bar up front keeps drive-by submissions from bouncing off CI.

## states
**Decision:** empty search → "No loops match — clear the filters" with a reset button. Catalog fetch failure → a single quiet retry link. Unparseable loops **cannot appear**: catalog.json is generated by the validator — a loop that fails the bar never enters the artifact.
**Rationale:** the build-time gate is the real state-handling — the UI never needs a "broken loop" state because the pipeline can't produce one.

## brand
**Decision:** tutorial tokens verbatim — field #0b0d12, card #0e1118, line rgba(255,255,255,.1), teal #34e0c4 (accents/focus/done-when), category badge colors from the site palette (eng teal, eval orange #ffb45a, design pink #ff86c2, ops blue #84b6ff, content green #9be58f), Ubuntu + Ubuntu Mono, the gap-ring mark in the header next to "Loop market". Same header pattern as playground.html (back-link to the tutorial).
**Rationale:** the market must read as a loopflow.live room, not a bolt-on — same rule as the playground.

## a11y
**Decision:** cards are `<article>`; expand/copy/submit are real `<button>`/`<a>` elements; visible 2px teal focus ring on every interactive element; chips carry `aria-pressed`; expanded source region gets `aria-expanded` on its trigger; badge text contrast ≥ 4.5:1 (dark text on bright badges); full keyboard path: tab through chips → search → cards → buttons.
**Rationale:** we ship an accessibility-repair loop in this very catalog — the gallery had better pass its own scan.
