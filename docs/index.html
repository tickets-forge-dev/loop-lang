Stop babysitting the agent.

LoopFlow is a small plain-English language for self-correcting AI coding workflows.

Write the loop once: goal, context, permissions, checks, human gates, and retry limit. The agent plans, edits, observes the real result, reflects on failure, and stops only when verified.

Try /loopflow exampleRead the docsOpen playground
loop "fix checkout tax":
  goal: checkout shows the right tax
  done when "pnpm test checkout" passes

  look at: src/checkout/, tests/checkout/, and the last failure
  allow edits automatically, but ask me before migrations

  each cycle: plan, then act, then observe
  when it fails: reflect, then plan again
  after 6 tries: stop and warn "stuck"

A loop is a visible control structure around an agent: goal → context → check → retry → stop.

Why LoopFlow?

Because the annoying part of AI coding is rarely the first prompt. It is everything after it: checking, reviewing, copying failures back into chat, and asking the agent to try again.

PRs keep bouncing

Review finds missing tests, weak error handling, or an API contract mismatch. A loop feeds that bar back into the agent and reruns the checks.

Features are “done” before they are proven

You still have to say: run the test, check the API, try curl, inspect the response, fix what failed. Put those checks in the loop.

Bug fixes stop too early

The agent says “fixed” but the failing test is still red. LoopFlow stops on the command, not on the model’s confidence.

Risky work needs a person

Let the agent edit code automatically, but require a human before migrations, deploys, payment changes, or pushes.

What should I do immediately?

Start with one workflow you already repeat manually. For example: “make this PR pass review,” “finish this feature and prove the API works,” or “fix this bug until the test is green.”

  1. Name the goal.
  2. List the files or docs the agent should look at.
  3. Write the real check: a test, command, curl script, scanner, or human review.
  4. Say what needs human approval.
  5. Add a retry limit so it cannot thrash forever.

Then run it and let the agent repeat the boring part for you.

Quick start

In Claude Code, ask LoopFlow for a starter file:

/loopflow example

Save the generated loop, edit the goal and check, then run it:

/loopflow run fix-checkout.loop

Prefer a terminal?

npm i -g @loop-lang/loop @loop-lang/runtime
loop init
loop-run show fix-checkout.loop
loop-run run fix-checkout.loop

The loop primitive

plan→act→observe→reflect↺

LoopFlow makes the control structure explicit. The model does the work; your machine runs the checks.

LineWhat it controls
goal:What success means in plain language.
look at:The context the agent reads before acting.
done whenThe finish line: a shell command, test, eval skill, or human confirmation.
allow… ask me before…What can run unattended and what must pause.
when it fails:The back-edge: reflect on the failure, then plan again.
after N tries:The thrash guard.

Real examples

PR review loop

loop "make PR reviewable":
  goal: the PR passes tests and addresses reviewer feedback
  done when "pnpm test" passes
  done when the skill "code-review" approves on the trajectory
    the bar: no weakened tests; reviewer comments addressed; no unrelated refactor
  look at: the PR diff, reviewer comments, src/, tests/, and the last failure
  each cycle: plan, then act, then observe
  when it fails: reflect on the strongest review objection, then plan again
  after 5 tries: stop and warn "review loop stuck"

API feature loop

loop "ship user search API":
  goal: the search endpoint returns filtered users with stable JSON
  done when "pnpm test api/search" passes
  done when "./scripts/curl-search-smoke.sh" passes
  look at: api/search/, tests/api/search/, docs/api.md, and the last failure
  allow edits automatically, but ask me before schema migrations
  each cycle: plan, then act, then observe
  when it fails: reflect on the failed test or curl output, then plan again
  after 6 tries: stop and warn "API behavior still not proven"

Bug fix loop

loop "fix billing apostrophe bug":
  goal: company names with apostrophes save correctly
  done when the test "billing.spec.ts::apostrophe" passes
  look at: billing/form.tsx, api/settings.ts, tests/billing/, and the last failure
  each cycle: plan, then act, then observe
  when it fails: reflect on which layer broke, then plan again
  after 6 tries: stop and warn "billing bug is thrashing"

Skills

Skills are reusable capabilities the loop may call while planning or acting. Pick how much freedom the loop has to add them.

skills: auto                         # discover/add useful skills with minimum friction
skills: ask                          # recommend additions, ask before adding
skills: fixed, seo-audit             # use only these explicit baseline skills
skills: none                         # use no skills
skills: auto, seo-audit              # start with seo-audit, auto-add more if useful

auto can add safe, useful skills. ask pauses before adding. fixed never adds beyond the named baseline. none disables skills.

Not magic. A control structure.

No fake done

done when runs a real command or asks a real reviewer.

No hidden checklist

The goal, context, permissions, checks, and stopping rules live in a file you can review.

No unbounded agent

Every serious loop should have after N tries.

No replacement for tests

LoopFlow makes your tests, scanners, curls, and reviews part of the workflow.

FAQ

What is an AI coding loop?

A workflow where an agent plans, edits, checks, reflects on failure, and retries until a real check passes or a guard stops it.

How is this different from prompting?

A prompt asks once. A loop keeps going until verified, with explicit context, gates, and stopping rules.

What tools does it work with?

Claude Code via /loopflow, headless via loop-run, and plain text anywhere.
